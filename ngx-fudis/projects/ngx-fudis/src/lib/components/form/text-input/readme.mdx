# Text Input

Basic text-input component with different type options: `email`, `number`, `password`, `tel`, `text` and `url`. Defaults to `text`.

Additional features include possibility to add tooltip to the label and maximum character indicator to the bottom right corner.

## Usage Guidelines

### Mandatory Properties

Necessary properties for text-input are `label` and `control`.

### Validators and Error handling

Fudis form components use Fudis Validators. See the full documentation on [how to use Fudis Validators](/docs/utilities-validators--documentation). 

If a custom validation is used, read [how to add Error message](/docs/components-form-error-message--documentation).

**Text Input validators**
- All of the Fudis Validators can be assigned to Text-input component.

#### Text Input maxLength indicator

[MaxLength](/docs/utilities-validators--documentation&args=#minlength-and-maxlength-validators) validator on Text Input does not prompt any error message but disable user to input more than allowed length of characters to the input field.
Additionally, when maxLength validation is used, by passing maxLength Input <code>[maxLength]="maxLength"</code> to Text Input, will activate the maximum character indicator to the bottom right corner.

#### Text Input as number input

Text-input [Min and max](/docs/utilities-validators--documentation&args=#min-and-max-validators) validators have additional <code>[minNumber]="minNumber"</code> and <code>[maxNumber]="maxNumber"</code> attributes for passing wanted min and max values to the text-input.

## Accessibility

- Label and guidance are linked to the text-input via id
- Screen reader is informed when maximum character length has 5 characters left and when maximum character count is reached
